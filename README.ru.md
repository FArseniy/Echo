# echo

> Временная комната. Приватная ссылка. Разговор, который исчезает после выхода.

[Открыть Echo](https://echo.erised.click/) | [English README](README.md) | [Дизайн-система](DESIGN_SYSTEM.md) | [Переменные окружения](.env.example)

```text
                         E C H O
                временные разговоры по ссылке

          +------------- комната -------------+
          |                                  |
          |   создать -- передать -- общаться |
          |                                  |
          +----------------------------------+
                    |                  |
              приватный P2P      групповая комната
              браузер <-> браузер    через Echo
```

Echo — чат без регистрации для быстрого разговора. Создайте комнату, передайте ссылку и PIN отдельно, а когда разговор закончен — закройте комнату. Без аккаунтов, постоянных профилей и постоянной истории.

## Выберите маршрут

| Режим | Подходит для | Как идут сообщения | История |
| --- | --- | --- | --- |
| **Приватный P2P** | Двух участников с прямым соединением | Браузер к браузеру | Не хранится |
| **Приватный P2P + TURN** | Двух участников, которым может понадобиться резервный маршрут | Сначала напрямую, при необходимости — зашифрованный TURN | Не хранится |
| **Групповой через сервер** | Комнат с несколькими участниками | Socket.IO через Echo | В памяти до закрытия комнаты |

### Приватный значит приватный

В приватных P2P-режимах Echo участвует только во входе в комнату, списке участников и WebRTC-сигналинге. Текст сообщений не передаётся через Node.js, Socket.IO и временную историю Echo.

WebRTC уже шифрует транспорт. Echo дополнительно создаёт в браузерах временные ключи ECDH P-256 и шифрует сообщения AES-256-GCM. Участники могут сравнить показанный код проверки вне чата.

> Важно: прямой P2P не означает анонимность. Собеседник потенциально может увидеть ваш сетевой адрес. TURN — резервный relay-маршрут, но не гарантия анонимности.

## Возможности

- 8-символьные коды комнат и PIN из 4-8 цифр;
- bcrypt-хеширование PIN: без PIN в URL, `localStorage` и серверных логах;
- временные имена, список участников и передача роли владельца;
- сообщения в реальном времени, индикатор набора и статусы отправки/доставки;
- краткое восстановление сессии через `sessionStorage`;
- лимиты PIN-попыток, сообщений и создания комнат;
- автоматическое удаление пустых и неактивных комнат;
- Helmet, строгий Origin для Socket.IO, HTTPS и WebSocket-ready Nginx.

- Задайте `PUBLIC_URL`, `CLIENT_ORIGIN`, `TURN_HOST` и `TURN_SHARED_SECRET` в `.env`.
- Не коммитьте `.env`, TURN-учётные данные, PIN комнат и токены сессий.
- После развёртывания убедитесь, что `/healthz` отвечает `{"status":"ok"}`.
